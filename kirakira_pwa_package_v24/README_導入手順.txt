きらきら作例シミュレーター PWA配布パッケージ
================================================

内容
----
- index.html : 店頭/配布用アプリ本体。元の v23 をベースに、PWA対応と「店員に見せる」メモを追加。
- manifest.webmanifest : ホーム画面追加用のアプリ情報。
- sw.js : 一度開いた後にオフライン起動しやすくするキャッシュ処理。
- icons/ : PWA用アイコン。
- event_qr_card_template.html : イベント掲示用QRカードの雛形。

推奨運用
--------
1. GitHub Pages 等の HTTPS 静的ホスティングに、このフォルダの中身を置く。
2. 公開URLをスマホ/タブレットで開く。
3. ホーム画面に追加する。
4. 一度アプリを起動する。
5. 機内モードにして、ホーム画面アイコンから起動できるか確認する。
6. イベント当日は店頭タブレットをホーム画面アイコンから起動する。

注意
----
- index.html を直接 file:// で開いても基本機能は動くが、PWAインストールと Service Worker のオフラインキャッシュは効かない。
- PWAとして配る場合は HTTPS または localhost で開く必要がある。
- QR配布する場合、初回アクセスには通信が必要。初回後はキャッシュによりオフライン起動しやすくなる。
- 端末やブラウザの省メモリ処理でキャッシュが消える可能性はあるので、イベント前に必ず機内モードで確認する。

GitHub Pagesでの最短手順
------------------------
1. GitHubで新規リポジトリを作る。
2. このフォルダの中身をリポジトリ直下にアップロードする。
3. Settings > Pages > Build and deployment で、Sourceを Deploy from a branch にする。
4. Branch を main / root にする。
5. 表示された https://...github.io/... のURLを開く。
6. event_qr_card_template.html にURLとQRを入れて印刷する。

店頭での導線
------------
1. 子供が「かたちを作る」で点を動かす。
2. 「つぎ：くるっと見る」を押す。
3. コピー数、図形色、中央ビジューを選ぶ。
4. 「スタート」を押す。
5. 「店員に見せる」を押してメモを表示する。
6. メモ画面または完成画面をスクショして、セミオーダー相談に使う。
